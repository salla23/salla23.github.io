BEV Perception with Temporal Fusion (in progress)

Camera-to-BEV segmentation · per-cell temporal filtering · github ↗

Bird's-eye-view semantic segmentation from surround cameras, with a temporal layer that fuses each frame's prediction into a persistent BEV grid using ego-motion. The thread connecting it to my occupancy-mapping work is the same one I keep pulling on: a single frame is noisy and partial, but the right Bayesian update across frames recovers a stable picture that no single frame contains.

Problem

Lifting camera images into a top-down BEV grid is the representation most planning stacks actually consume, but per-frame BEV segmentation flickers — a cell flips between classes as objects are occluded, enter at range, or pass through shadow and glare. Planning wants temporal stability, not a fresh guess every frame.

Approach

A pretrained lift-splat model produces per-frame BEV class logits on nuScenes. On top of it, a per-cell filter carries each cell's class estimate and its uncertainty forward in time: ego-motion warps the previous grid into the current frame, and each new prediction updates the estimate with a confidence weight rather than overwriting it. It is the same log-odds occupancy idea generalized from a static two-class map to a moving-frame, many-class one — the Kalman gain plays the role the fixed log-odds increment plays in the occupancy grid.

Evaluation plan

Per-class IoU on the validation split, latency on stated hardware, and a failure analysis across the three regimes that break BEV in practice — long range, night, and occlusion — with a before/after that quantifies what the temporal layer buys in both accuracy and flicker.
